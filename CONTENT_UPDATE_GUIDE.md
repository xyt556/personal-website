# 交互式网站内容更新指南

## 🎯 功能说明

你的网站已经配置了完整的**交互式内容管理系统**，可以让你在浏览器中可视化编辑网站内容，然后一键部署更新。

---

## 📝 使用步骤

### 第一步：启动本地开发服务器

```bash
npm run dev
```

访问 http://localhost:5173

### 第二步：进入管理后台

1. 点击网站右下角的**齿轮图标** ⚙️
2. 输入管理员密码：`admin123`
3. 点击"登录"

### 第三步：编辑网站内容

在管理后台，你可以编辑：

- **首页横幅**：姓名、职位、简介、头像、社交链接
- **软件作品**：项目名称、描述、技术栈、链接
- **关于我**：个人介绍、统计数据、亮点特色
- **技术栈**：技能分类、技能名称、熟练度
- **联系方式**：邮箱、地址、工作时间

编辑完成后，点击右上角的**"保存更改"**按钮。

### 第四步：导出配置文件

点击右上角的**"导出配置"**按钮，会下载一个 `site-data.json` 文件。

### 第五步：更新网站代码

将下载的 `site-data.json` 文件移动到项目根目录（与 package.json 同级）。

然后选择以下方式之一：

#### 方式一：自动部署（推荐）

双击运行 `deploy-update.bat` 文件，脚本会自动：
1. 更新代码
2. 构建项目
3. 提交到 Git
4. 推送到远程仓库

#### 方式二：手动操作

```bash
# 1. 更新代码
npm run update-content

# 2. 预览更改
npm run dev

# 3. 提交并推送
git add .
git commit -m "更新网站内容"
git push
```

### 第六步：等待自动部署

推送后，Vercel 会在 1-2 分钟内自动完成部署。你的网站内容已更新！

---

## 🔐 修改管理员密码

编辑 `src/store/siteData.ts` 文件，找到：

```typescript
const ADMIN_PASSWORD = 'admin123';
```

修改为你自己的密码。

---

## 💡 提示

- 管理后台的更改会保存在浏览器本地存储中
- 点击"重置"可以恢复到默认内容
- 导出的配置文件包含所有网站内容
- 每次更新后记得推送到 Git 仓库

---

## 🆘 常见问题

**Q: 为什么导出的配置没有生效？**
A: 确保 `site-data.json` 文件放在项目根目录，然后重新运行 `npm run update-content`。

**Q: 如何在手机上编辑？**
A: 访问网站，点击右下角齿轮图标即可进入管理后台。但建议在电脑上操作，体验更好。

**Q: 可以撤销更改吗？**
A: 可以！Git 会保存所有历史版本。使用 `git log` 查看历史，`git reset` 回退版本。

**Q: 部署后多久能看到更新？**
A: 通常 1-2 分钟。可以在 Vercel 控制台查看部署进度。
