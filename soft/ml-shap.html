软件简介

机器学习及SHAP模型解释工具 是一款功能强大的机器学习建模与模型解释软件，集成了数据预处理、特征工程、模型训练、参数优化、模型评估、SHAP解释、LIME解释等完整的数据科学工作流程。

主要特性

  • ✅ 支持多种数据格式：CSV、Excel (.xlsx/.xls)
  • ✅ 丰富的算法库：集成30+种机器学习算法
  • ✅ 智能参数优化：网格搜索、随机搜索、贝叶斯优化
  • ✅ 完整的SHAP分析：支持10+种SHAP可视化图表
  • ✅ LIME可解释性：局部解释分析
  • ✅ AutoML自动建模：一键自动选择最优模型
  • ✅ 多格式导出：Excel、CSV、PDF、SVG、PNG等
  • ✅ 项目配置管理：保存/加载项目配置，快速恢复工作状态

系统要求

操作系统

  • Windows 10/11 (推荐)
  • macOS 10.14+
  • Linux (Ubuntu 18.04+)

硬件要求

  • CPU: 双核及以上 (推荐四核或更多)
  • 内存: 4GB RAM (推荐8GB或更多)
  • 硬盘: 至少500MB可用空间

软件依赖

  • Python 3.7+
  • 必需库：numpy, pandas, scikit-learn, matplotlib, tkinter
  • 可选库：xgboost, lightgbm, catboost, shap, lime, imbalanced-learn

快速入门

5分钟快速上手

  1. 启动软件：双击运行 shap_analyzer_v5.7.6_fig_en.py
  2. 加载数据：点击"步骤1"中的"选择数据"按钮，选择您的CSV或Excel文件

详细功能说明

步骤1: 数据加载

功能说明

支持两种数据加载模式：

  1. 单文件模式：加载一个数据文件，自动按比例划分训练集和测试集
  2. 分别加载模式：分别加载训练集和测试集文件

操作步骤

模式一：单文件加载

  1. 点击"📂 选择数据 (CSV/Excel)"按钮
  2. 在文件选择对话框中选择数据文件
  3. 软件会自动读取并显示数据基本信息

模式二：分别加载训练集/测试集

  1. 勾选"分别加载训练集/测试集"选项
  2. 点击"📂 加载训练集文件"选择训练数据
  3. 点击"📂 加载测试集文件"选择测试数据
  4. 软件会自动检查训练集和测试集的列一致性

支持的文件格式

  • CSV文件 (.csv)
  • Excel文件 (.xlsx, .xls)

注意事项

  • 数据文件第一行应为列名
  • 确保数据中包含目标变量列
  • 训练集和测试集应具有相同的列结构

步骤2: 选择目标列

功能说明

指定要预测的目标变量（因变量）。

操作步骤

  1. 在"目标变量"下拉框中选择要预测的列
  2. 软件会自动将该列从特征列中排除

注意事项

  • 目标列会自动从特征列中移除，避免数据泄露
  • 切换目标列时，软件会自动更新特征列选择状态

步骤3: 选择特征列

功能说明

选择用于训练模型的特征变量（自变量）。

操作步骤

  1. 在特征列表中勾选要使用的特征
  2. 使用"✓ 全选"快速选择所有特征
  3. 使用"✗ 全不选"取消所有选择

注意事项

  • 目标列会被自动排除
  • 建议移除高度相关的特征
  • 常数特征（方差为0）应被移除

步骤4: 训练集比例设置

功能说明

设置训练集和测试集的划分比例。

参数说明

  • 训练集比例：默认70%，可调节范围50%-90%
  • 随机数种子：默认42，相同种子可复现相同的数据划分

推荐设置

  • 小数据集（≤1000样本）：70%-80%
  • 大数据集（>1000样本）：80%-90%

注意事项

  • 此设置仅在"单文件加载"模式下有效
  • "分别加载训练集/测试集"模式会跳过此步骤

步骤5: 缺失值处理

功能说明

处理数据中的缺失值，提供多种填充策略。

参数设置

1. 启用缺失值处理

  • 勾选"启用缺失值处理"开关

2. 按缺失率剔除列

  • 勾选"按缺失率剔除列（>阈值）"
  • 设置阈值（0-1之间，默认0.5表示50%）
  • 缺失率超过阈值的列将被删除

3. 数值列填充策略

  • 均值：使用列均值填充（推荐）
  • 中位数：使用列中位数填充
  • 众数：使用列众数填充
  • 常数：使用指定常数填充
  • 前向填充：使用前一个有效值填充
  • 后向填充：使用后一个有效值填充
  • 多重插补(MICE)：使用链式方程多重插补（高级）

4. 分类列填充策略

  • 众数：使用列众数填充（推荐）
  • 常数：使用指定常数填充
  • 前向填充：使用前一个有效值填充
  • 后向填充：使用后一个有效值填充

5. MICE参数（仅当选择"多重插补(MICE)"时生效）

  • 插补次数M：默认5次，M>1时对多次插补结果取平均
  • 迭代上限：默认10次

高级功能

  • 保存填充后的数据集：导出处理后的完整数据
  • 保存编码前数据：仅保存缺失填充后的数据（未编码）

注意事项

  • 预测/新数据会自动复用训练阶段计算的填充值，保证处理一致性
  • MICE方法计算量较大，适用于高精度需求场景

步骤6: 分类变量编码

功能说明

将分类变量转换为数值形式，以便机器学习模型处理。

编码方法

1. Label编码(整数)

  • 将每个类别映射为一个整数
  • 适用于有序分类变量
  • 计算速度快，内存占用小

2. One-Hot编码

  • 将每个类别展开为二进制列
  • 适用于无序分类变量
  • 可设置"最大类别数"限制（默认30）
  • 超过限制的类别会被归为"其他"类

3. 频率编码

  • 使用类别出现频率作为编码值
  • 适用于高基数分类变量
  • 保持数据维度不变

4. 目标均值编码

  • 使用目标变量的均值进行编码
  • 适用于分类问题
  • 可设置平滑参数（默认10.0）
  • 注意：可能导致数据泄露，需谨慎使用

推荐选择

  • 低基数分类变量（<10个类别）：One-Hot编码
  • 高基数分类变量（≥10个类别）：频率编码或目标均值编码
  • 有序分类变量：Label编码

步骤7: 数据标准化

功能说明

对数值特征进行标准化或归一化处理。

标准化方法

1. 不标准化

  • 保持原始数据不变
  • 树模型（随机森林、XGBoost等）不需要标准化

2. Z-score标准化

  • 转换为均值为0、标准差为1的分布
  • 公式：(x - μ) / σ
  • 适用于线性模型、SVM、神经网络

3. Min-Max归一化

  • 缩放到[0, 1]区间
  • 公式：(x - min) / (max - min)
  • 适用于需要固定范围的特征

推荐选择

  • 树模型：不需要标准化
  • 线性模型、SVM、神经网络：推荐Z-score标准化
  • KNN：必须标准化

步骤8: 数据不平衡处理

功能说明

处理分类问题中的类别不平衡问题。

处理方法

1. 不处理

  • 保持原始数据分布

2. 过采样方法

  • SMOTE：合成少数类过采样技术（推荐）
  • ADASYN：自适应合成采样
  • BorderlineSMOTE：边界SMOTE
  • SVMSMOTE：基于SVM的SMOTE
  • RandomOverSampler：随机过采样

3. 欠采样方法

  • RandomUnderSampler：随机欠采样
  • EditedNearestNeighbours：编辑最近邻
  • TomekLinks：Tomek链接

4. 混合方法

  • SMOTETomek：SMOTE + Tomek链接
  • SMOTEENN：SMOTE + 编辑最近邻

参数设置

  • 采样比例：默认1.0（完全平衡）
  • k_neighbors：SMOTE的近邻数（默认5）

注意事项

  • 仅适用于分类问题
  • 过采样可能导致过拟合
  • 欠采样可能丢失重要信息
  • 推荐先尝试SMOTE

步骤9: SHAP样本设置

功能说明

设置用于SHAP分析的样本数量。

参数设置

  • SHAP样本百分比：默认10%，可调节范围1%-100%
  • KernelExplainer背景样本数：默认100，范围20-500

推荐设置

  • 小数据集（≤1000样本）：50%-100%
  • 中等数据集（1000-10000样本）：10%-30%
  • 大数据集（>10000样本）：5%-10%

注意事项

  • 样本数越多，SHAP计算越准确，但计算时间越长
  • KernelExplainer计算速度较慢，建议使用较小的背景样本数

步骤10: 问题类型

功能说明

指定问题是分类问题还是回归问题。

选项说明

  • 自动检测：根据目标变量的类型自动判断（推荐）
  • 分类问题：预测离散类别
  • 回归问题：预测连续数值

自动检测规则

  • 目标变量为字符串类型 → 分类问题
  • 目标变量为数值类型且唯一值≤10 → 分类问题
  • 目标变量为数值类型且唯一值>10 → 回归问题

步骤11: 选择算法

功能说明

选择要训练的机器学习算法，支持多选。

可用算法

🌟 集成学习 - 推荐

  • 随机森林 (RandomForest) ⭐推荐 - 鲁棒性强，适合大多数场景
  • XGBoost ⭐推荐 - 性能优异，竞赛常用
  • LightGBM ⭐推荐 - 速度快，适合大数据
  • CatBoost ⭐推荐 - 自动处理分类特征
  • 梯度提升 (GradientBoosting) - 经典集成方法
  • 直方图梯度提升 (HistGradientBoosting) - 快速梯度提升
  • AdaBoost - 自适应提升
  • 极端随机树 (ExtraTrees) - 随机性更强
  • Bagging集成 - 并行集成方法

📏 线性模型

  • 逻辑/线性回归 (LogisticReg/LinearReg) - 基础模型
  • 岭回归 (Ridge) - L2正则化
  • Lasso回归 - L1正则化
  • 弹性网络 (ElasticNet) - L1+L2正则化
  • 随机梯度下降 (SGD) - 大规模数据

🌴 树模型

  • 决策树 (DecisionTree) - 可解释性强

🎪 支持向量机

  • SVM - 适合中小数据集

👥 K近邻

  • KNN - 简单直观

🎲 贝叶斯分类（仅分类）

  • 高斯朴素贝叶斯 (GaussianNB)
  • 伯努利朴素贝叶斯 (BernoulliNB)
  • 补集朴素贝叶斯 (ComplementNB)

📊 判别分析（仅分类）

  • 线性判别分析 (LDA)
  • 二次判别分析 (QDA)

🧠 神经网络

  • 多层感知机 (MLP) - 深度学习基础

推荐选择

  • 初学者：随机森林
  • 追求性能：XGBoost、LightGBM、CatBoost
  • 可解释性：决策树、线性回归
  • 大数据：LightGBM、SGD

步骤12: 自定义模型参数

功能说明

为特定算法设置自定义参数，覆盖默认值。

操作步骤

  1. 在"选择算法"下拉框中选择要配置的算法
  2. 在文本框中输入JSON格式的参数
  3. 点击"💾 保存"保存参数配置

JSON格式示例

{
  "n_estimators": 100,
  "max_depth": 5,
  "learning_rate": 0.1,
  "min_samples_split": 2,
  "min_samples_leaf": 1
}

常用参数说明

随机森林 (RandomForest)

  • n_estimators: 树的数量（默认100）
  • max_depth: 树的最大深度（默认None）
  • min_samples_split: 分裂所需最小样本数（默认2）
  • min_samples_leaf: 叶节点最小样本数（默认1）

XGBoost

  • n_estimators: 提升轮数（默认100）
  • max_depth: 树的最大深度（默认6）
  • learning_rate: 学习率（默认0.1）
  • subsample: 样本采样比例（默认1.0）

LightGBM

  • n_estimators: 提升轮数（默认100）
  • max_depth: 树的最大深度（默认-1）
  • learning_rate: 学习率（默认0.1）
  • num_leaves: 叶子节点数（默认31）

注意事项

  • 参数值类型必须正确（整数、浮点数、布尔值、字符串）
  • 字符串参数需要用引号括起来
  • 布尔值使用 true 或 false

步骤13: 参数优化

功能说明

自动搜索最优模型参数，提升模型性能。

优化方法

1. 无优化

  • 使用默认参数或自定义参数

2. 网格搜索

  • 遍历所有参数组合
  • 计算量大，适合小参数空间
  • 最全面但最慢

3. 随机搜索

  • 随机采样参数组合
  • 比网格搜索快
  • 适合大参数空间

4. 贝叶斯搜索 ⭐智能

  • 智能优化，根据历史结果选择参数
  • 效果好且速度快（推荐）
  • 需要安装 scikit-optimize

参数设置

优化专用参数

  • K折数：交叉验证折数（默认3）
  • 评分指标：优化目标（默认auto）
  • 尝试次数：随机/贝叶斯搜索的迭代次数（默认20）

搜索空间设置

  • 使用内置搜索空间：使用软件预设的参数范围
  • JSON完全替换内置：完全自定义参数范围
  • JSON与内置合并：在内置基础上添加自定义参数

对比选项

  • 勾选"优化后与当前默认/自定义参数对比"可同时评估优化前后的性能

JSON搜索空间示例

{
  "n_estimators": [50, 100, 200],
  "max_depth": [3, 5, 7, 10],
  "learning_rate": [0.01, 0.05, 0.1, 0.2]
}

注意事项

  • 参数优化会增加训练时间
  • 贝叶斯搜索需要安装 scikit-optimize
  • 建议先用较小的尝试次数（如10-20次）

步骤14: 交叉验证设置

功能说明

使用K折交叉验证评估模型性能，比单次划分更可靠。

参数设置

  • 启用交叉验证：勾选开关
  • 折数：默认5折，可设置3-20折
  • 评估指标：默认auto（自动选择）

评估指标选项

分类问题

  • accuracy: 准确率
  • precision: 精确率
  • recall: 召回率
  • f1: F1分数
  • roc_auc: ROC曲线下面积

回归问题

  • r2: R²决定系数
  • neg_mean_squared_error: 均方误差（负值）
  • neg_mean_absolute_error: 平均绝对误差（负值）

推荐设置

  • 小数据集：5-10折
  • 大数据集：3-5折
  • 折数越多，评估越可靠，但计算时间越长

步骤15: 并行计算设置

功能说明

设置并行计算参数，加速模型训练。

参数设置

  • 并行作业数 (n_jobs)：默认自动检测

    • -1: 使用所有CPU核心
    • 1: 单线程（最稳定）
    • 2-4: 使用指定数量的核心

  • 临时放开BLAS线程：训练时增加数值计算线程数

推荐设置

  • 开发环境：使用多核（2-4核）
  • 打包环境：单线程或2核（更稳定）

注意事项

  • 并行计算会增加内存使用
  • 某些算法不支持并行
  • 打包环境建议使用较少的核心数

步骤16: 训练模型

功能说明

开始训练选定的模型。

训练方式

1. 开始训练

  • 训练所有选中的算法
  • 显示训练进度和日志
  • 训练完成后显示模型对比结果

2. AutoML自动建模

  • 自动尝试多种算法
  • 自动调参
  • 自动选择最优模型
  • 一键完成建模流程

训练流程

  1. 数据预处理（缺失值、编码、标准化）
  2. 数据划分（训练集/测试集）
  3. 数据不平衡处理（如启用）
  4. 模型训练
  5. 参数优化（如启用）
  6. 模型评估
  7. 结果展示

训练结果

  • 模型性能对比表：显示所有模型的评估指标
  • 最佳模型推荐：自动推荐最优模型
  • 训练日志：详细记录训练过程

步骤17: 模型评估

功能说明

详细评估训练好的模型，生成评估报告和图表。

评估指标

分类问题

  • 准确率 (Accuracy)
  • 精确率 (Precision)
  • 召回率 (Recall)
  • F1分数
  • ROC-AUC
  • 混淆矩阵
  • 分类报告

回归问题

步骤18: 特征选择

功能说明

自动选择最重要的特征，降低模型复杂度。

特征选择方法

1. 过滤法

  • 方差阈值：移除低方差特征
  • 相关系数：移除高相关特征
  • 卡方检验：选择与目标最相关的特征
  • 互信息：基于信息增益选择特征

2. 包装法

  • 递归特征消除 (RFE)：递归地移除最不重要的特征
  • RFECV：带交叉验证的RFE，自动确定最优特征数

3. 嵌入法

  • 基于模型的重要性：使用模型内置的特征重要性
  • SelectFromModel：根据重要性阈值选择特征

操作步骤

  1. 选择特征选择方法
  2. 设置参数（如保留特征数）
  3. 点击"应用特征选择"
  4. 查看特征选择结果
  5. 可选择"重置特征选择"恢复原始特征

注意事项

  • 特征选择后会更新训练数据
  • 建议在训练模型后再进行特征选择
  • 特征选择可能影响模型性能，需谨慎使用

步骤19: SHAP分析

功能说明

使用SHAP（SHapley Additive exPlanations）方法解释模型预测。

SHAP值计算

1. 计算SHAP值

  • 点击"计算SHAP值"按钮
  • 根据样本数量，计算时间可能较长
  • 计算完成后自动缓存结果

2. SHAP交互值

  • 计算特征间的交互效应
  • 计算量较大，建议小样本使用

SHAP可视化图表

1. 摘要图 (Summary Plot)

  • 显示所有特征的重要性排序
  • 每个点代表一个样本
  • 颜色表示特征值大小
  • 横轴表示SHAP值

2. 条形图 (Bar Plot)

  • 显示特征重要性排序
  • 基于平均绝对SHAP值

3. 依赖图 (Dependence Plot)

  • 显示单个特征与SHAP值的关系
  • 自动选择交互特征
  • 可手动指定特征

4. 交互图 (Interaction Plot)

  • 显示两个特征的交互效应
  • 颜色表示交互特征值

5. 瀑布图 (Waterfall Plot)

  • 显示单个样本的预测解释
  • 从基准值到预测值的贡献分解

6. 力图 (Force Plot)

  • 显示单个样本的特征贡献
  • 可视化正负贡献

7. 决策图 (Decision Plot)

  • 显示预测路径
  • 适合多特征模型

8. 热力图 (Heatmap)

  • 显示所有样本的SHAP值分布
  • 聚类相似样本

9. PDP图 (Partial Dependence Plot)

  • 显示特征对预测的平均效应
  • 不依赖SHAP值

10. ICE图 (Individual Conditional Expectation)

  • 显示每个样本的预测变化
  • PDP的扩展

11. ALE图 (Accumulated Local Effects)

  • 更准确的特征效应估计
  • 不受特征相关性影响

多分类问题

  • 可切换不同类别查看SHAP值
  • 每个类别有独立的SHAP解释

图表交互

  • 右键菜单：全屏显示、保存图表
  • 支持格式：PNG、JPG、SVG、PDF、EPS、TIF
  • 字体设置：可自定义字体和大小

常见问题

Q1: 软件启动很慢怎么办？

A: 软件启动时需要加载多个机器学习库，首次启动可能需要10-30秒。后续启动会更快。如果仍然很慢，可以检查：

  • 是否安装了过多不必要的库
  • 计算机性能是否满足要求
  • 是否有杀毒软件干扰

Q2: 加载数据时提示编码错误怎么办？

A: 软件会自动尝试UTF-8和GBK编码。如果仍然失败，可以：

  • 用Excel打开CSV文件，另存为UTF-8编码
  • 使用文本编辑器（如Notepad++）转换编码
  • 检查文件是否损坏

Q3: 训练模型时内存不足怎么办？

A: 可以尝试：

  • 减少训练样本数量（步骤9设置SHAP样本百分比）
  • 减少特征数量（步骤18特征选择）
  • 选择计算量较小的算法（如线性模型）
  • 关闭其他占用内存的程序
  • 增加计算机内存

Q4: SHAP计算时间太长怎么办？

A: SHAP计算复杂度较高，可以：

  • 减少样本数量（步骤9设置SHAP样本百分比）
  • 使用TreeExplainer支持的模型（随机森林、XGBoost等）
  • 避免使用KernelExplainer（速度最慢）
  • 使用缓存功能，避免重复计算

Q5: 如何选择合适的算法？

A: 算法选择建议：

  • 初学者：随机森林（鲁棒性强，参数少）
  • 追求性能：XGBoost、LightGBM、CatBoost
  • 大数据：LightGBM、SGD
  • 可解释性：决策树、线性回归
  • 不确定：使用AutoML自动选择

Q6: 参数优化有必要吗？

A: 参数优化可以提升模型性能，但会增加训练时间。建议：

  • 默认参数通常已经不错，可以先不优化
  • 如果默认效果不理想，再尝试参数优化
  • 优先使用贝叶斯搜索（效率最高）
  • 从较小的尝试次数开始（如10-20次）

Q7: 如何解释SHAP图表？

A: SHAP图表解读：

  • 摘要图：特征重要性排序，颜色表示特征值大小
  • 瀑布图：单个样本的预测解释，显示各特征贡献
  • 依赖图：特征值与SHAP值的关系，显示特征如何影响预测
  • SHAP值：正值表示增加预测值，负值表示降低预测值
  • SHAP值绝对值：表示特征重要性

Q8: 分类变量编码如何选择？

A: 编码方法选择：

  • 低基数（<10个类别）：One-Hot编码
  • 高基数（≥10个类别）：频率编码或目标均值编码
  • 有序分类：Label编码
  • 不确定：尝试多种方法，选择效果最好的

Q9: 数据不平衡如何处理？

A: 不平衡处理建议：

  • 首先检查不平衡程度（类别比例）
  • 轻度不平衡（比例>1:5）：可以不处理
  • 中度不平衡（比例1:5-1:10）：尝试SMOTE
  • 重度不平衡（比例<1:10）：尝试SMOTEENN或SMOTETomek
  • 同时调整类别权重（在模型参数中设置）

Q10: 如何导出结果用于论文？

A: 论文图表导出建议：

  • 矢量图：SVG或PDF格式（无损缩放）
  • 位图：PNG格式，DPI设置为300或600
  • 图表字体：设置为论文要求的字体
  • 颜色方案：考虑黑白打印，使用不同线型或标记
  • 图表大小：根据论文版面调整
